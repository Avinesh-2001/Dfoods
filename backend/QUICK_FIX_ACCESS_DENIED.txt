# ⚡ QUICK FIX: OAuth Access Denied Error

## The Problem
Your OAuth app is in "Testing" mode, so only approved test users can access it.

## The Solution (Takes 2 Minutes)

### Step 1: Open Google Cloud Console
Direct link: https://console.cloud.google.com/apis/credentials/consent?project=dfoods-477119

### Step 2: Add Test User
1. Scroll down to the "Test users" section
2. Click the "+ ADD USERS" button
3. Enter: avijangid7011@gmail.com
4. Click "ADD"
5. Click "SAVE" at the bottom

### Step 3: Wait 1-2 Minutes
Changes take a moment to propagate.

### Step 4: Try Again
Use the same authorization URL:
https://accounts.google.com/o/oauth2/v2/auth?access_type=offline&scope=https%3A%2F%2Fwww.googleapis.com%2Fauth%2Fgmail.send&prompt=consent&response_type=code&client_id=435536199150-45qkuagjh5ou7hudabjestbgfbaff7nk.apps.googleusercontent.com&redirect_uri=https%3A%2F%2Fdfoods.onrender.com%2Foauth2callback

## That's It!

After adding yourself as a test user, you'll be able to authorize and get your refresh token.
